🎯 What I learned today  
• Set up Tailwind CSS in a Vite project for lightning-fast development ⚡  
• Understood the utility-first philosophy — styling directly in HTML with predefined classes 🧩  
• Explored core concepts: spacing, typography, colors, borders, shadows, and flex/grid layout 🎨📐  
• Practiced responsive design using Tailwind’s mobile-first breakpoints 📱💻  
• Customized themes using `tailwind.config.js` for branding and consistency 🎛️  
• Used hover, focus, and active states for interactive UI components 🎯🖱️  
• Built a simple responsive card layout with Tailwind classes 🧱📦

📘 Key Concepts Applied  
| Concept              | Example Class             | Purpose                          |
|----------------------|---------------------------|----------------------------------|
| Margin & Padding     | `m-4`, `p-2`              | Spacing around elements          |
| Typography           | `text-xl`, `font-bold`    | Font size and weight             |
| Colors               | `bg-blue-500`, `text-white` | Background and text colors     |
| Flexbox & Grid       | `flex`, `grid`, `gap-4`   | Layout and alignment             |
| Borders & Radius     | `border`, `rounded-lg`    | Element outlines and curves      |
| Shadows              | `shadow-md`               | Depth and elevation              |
| Responsive Design    | `md:text-lg`, `lg:p-6`    | Breakpoint-based styling         |

🧠 Key Takeaways  
• Tailwind speeds up development by reducing context-switching between HTML and CSS ⚡  
• Utility classes make styling predictable, scalable, and easy to debug 🧠🔍  
• Responsive design is intuitive with Tailwind’s breakpoint system 📱➡️🖥️  
• Customization via config files allows full control over design tokens 🎨🛠️  
• Tailwind encourages clean, component-based architecture with minimal CSS files 🧼📦
